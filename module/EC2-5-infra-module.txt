Without modules â†’ youâ€™ll keep copy-pasting the aws_instance block 5 times, just changing subnet/security group/instance_type.

With modules â†’ you write one EC2 blueprint (module), and then just call it 5 times with different inputs.



---

ðŸ”¹ Step 1: Create an EC2 Module

Make a folder modules/ec2/ and inside it:

modules/ec2/main.tf

resource "aws_instance" "this" {
  ami                         = var.ami_id
  instance_type               = var.instance_type
  subnet_id                   = var.subnet_id
  vpc_security_group_ids      = var.security_group_ids
  key_name                    = var.key_name
  associate_public_ip_address = var.associate_public_ip

  tags = {
    Name = var.instance_name
  }
}

modules/ec2/variables.tf

variable "ami_id" {}
variable "instance_type" {}
variable "subnet_id" {}
variable "security_group_ids" { type = list(string) }
variable "key_name" {}
variable "instance_name" {}
variable "associate_public_ip" { default = false }

modules/ec2/outputs.tf

output "instance_id" {
  value = aws_instance.this.id
}

output "public_ip" {
  value = aws_instance.this.public_ip
}

output "private_ip" {
  value = aws_instance.this.private_ip
}

ðŸ”¹ Step 2: Call the Module in main.tf

Now instead of writing 5 aws_instance blocks, you call the same module 5 times:
